# 中文输出

## 目标

让本项目中的生成内容默认符合中文工作语境，尤其是 PRD、Trellis 任务文档、需求分析、实现计划、验收标准、接口说明、最终总结和过程说明。

## 执行规则

1. 用户明确要求中文环境、中文输出或中文文档时，正文使用简体中文。
2. 保留代码标识符、文件路径、接口路由、字段名、枚举值、命令、提交信息示例中的英文原文。
3. 技术名词可按项目习惯中英混用，例如 `Repository`、`Template`、`ApplicationService`、Git provider。
4. PRD 和任务文档使用中文标题与中文段落，结构保持清晰：背景、目标、非目标、功能需求、流程、API 契约、数据模型、实现计划、验收标准、待确认问题。
5. 面向用户的阶段更新、最终回复和方案判断使用中文；除非用户指定英文或目标文件已有强制英文规范。
6. 如果系统或开发者级规范强制要求其他语言，遵守更高优先级规范；如果只是项目默认文档语言与用户当前中文要求冲突，在允许的范围内按用户当前请求处理，并在必要时简短说明。
7. 不翻译会破坏准确性的内容，例如 JSON key、DTO 类名、schema 字段名、错误类名、常量名和命令行参数。

## 检查清单

- 文档主体是否为简体中文。
- 路由、字段、类名、文件路径是否保持原文。
- 是否避免了中英混乱导致的字段名误译。
- 最终回复是否用中文说明改动结果。
